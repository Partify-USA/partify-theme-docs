Home Page

Purpose​

The homepage is vehicle-aware. Almost everything below the slideshow keys off the shopper's active vehicle (the Year/Make/Model stored in the garage) and reshapes itself around it — showing "Top Parts for your vehicle", vehicle-filtered category tiles, and a garage bar. With no vehicle set, the same sections fall back to generic prompts that push the shopper to pick a vehicle first.

The template is templates/index.json (auto-generated by the theme editor — expect it to be overwritten when sections are reordered in admin). Variants index.boost-sd-original.json and index.context.b2b.json exist for Boost SD and B2B contexts and are out of scope here.

Section Order (live)​

index.json lists many sections, several of which are disabled. The enabled, live order is:

  1. Scheduled slideshow (sections/scheduled-slideshow.liquid) — hero. Supports always-on slides plus date-windowed (e.g. Black Friday) and seasonal slides.
  2. Vehicle hero banner (sections/vehicle-hero-banner.liquid) — shows an image of the shopper's selected vehicle; collapses when no vehicle is set.
  3. Garage bar (sections/homepage-vehicle-hero.liquid) — the "Shopping for: {vehicle}" strip with a Change button.
  4. Category tabs (sections/homepage-visual-categories.liquid) — the CarID-style "Top Categories" tabs.
  5. Vehicle hub (sections/homepage-vehicle-hub.liquid) — the main "Top Parts for {vehicle}" grid + category tiles; loads assets/homepage-a-hub.js.
  6. Trust strip (sections/homepage-trust-strip.liquid).
  7. Reviews carousel — Yotpo app block.
  8. Why Partify (sections/homepage-why-partify.liquid) — value cards.

Disabled sections (general sales banner, dynamic collection list, testimonials, blog posts, site-review columns, an EasySearch app embed) remain in the JSON but do not render.

Dynamic vs Static Content​

  • Static: the slideshow, trust strip, Why-Partify cards, and the default category tabs are authored as section/blocks in Liquid.
  • Dynamic: when a vehicle is selected, assets/homepage-a-hub.js replaces the static category tiles and "Top Parts" grid with cards built at runtime from live product data for that vehicle, and rewrites each card's link to the correct filtered collection URL.

Data In​

  • localStorage.searchTerms — the garage; an array of vehicle objects. The first entry is the active vehicle. This is the single source of truth for "which vehicle." See Adding Search Terms.
  • Storefront GraphQL API — homepage-a-hub.js queries a collection's products and their custom.collection_types metafield + product_type to build category tiles. The Storefront token is currency-branched (window.shopCurrency), with separate tokens for the US and CA stores.
  • Collection products JSON — /collections/{handle}/products.json for the "Top Parts" grid, with a fallback chain (vehicle collection → top-sellers → all).
  • customer.metafields.garage.vehicles — the signed-in shopper's saved garage, merged into localStorage on sign-in.
  • window.shopCurrency / window.shopLocale — set in layout/theme.liquid.

Data Out / Side Effects​

  • Rewrites category/grid links to vehicle-filtered collection URLs (?filter.p.m.custom.collection_types=…&filter.p.product_type=…).
  • Opens the shared garage modal (window.toggleGaragePopup) when a shopper picks a category without a vehicle set; the pending category is stashed in sessionStorage and the shopper is redirected to the filtered collection after choosing a vehicle.
  • Pushes IntelliGems analytics events (popularPickClicked, topCategoryNavigated, viewAllPartsClicked, animatedGarageClicked).

Key Logic Areas​

Vehicle hero banner​

Reads the active vehicle, builds a Year/Make/Model collection handle, and fetches that collection's vehicle image. If the returned image looks like a part image (keyword test), it uses the section's fallback image instead. Hidden entirely when no vehicle is set.

Dynamic category tiles (the hub)​

For the active vehicle, queries products and groups them by custom.collection_types and product_type to build category cards, deduping by product type. "View all parts" points at the vehicle's collection (or /collections/all).

Known Constraints & Gotchas​

  • index.json is auto-generated — reordering sections in the theme editor can rewrite it.
  • The Storefront API tokens are hardcoded in homepage-a-hub.js and selected by currency.
  • Two readers of the vehicle can drift: vehicle-hero-banner.liquid reads searchTerms[0] directly, while homepage-a-hub.js prefers the canonical getGarageSearchTerms(). Heavy debouncing/polling exists specifically to keep them in sync — be careful changing it.
